01The problem

Human memory drifts mid-conversation — nobody lies on purpose.

A budget stated “firm at $40,000” becomes “$55,000” nine lines later, mid-negotiation, while everyone's thinking about something else. That's not a villain — it's how working memory fails under load.

A raw transcript doesn't fix this — you'd have to re-read the whole thing to catch the drift. A summary is worse: it flattens the exact tension you needed to see.

The cost compounds silently until it surfaces as an awkward “wait, that's not what we agreed” email — days later, once it's expensive to unwind.

02The insight

Four verbs, not one feature.

What's missing is something closer to a sharp chief of staff sitting in on the call: remembers every number anyone has said, notices the instant a new one doesn't match, and — because not everything said in a room should live forever — can be told to strike a line from the record.

remember

Every utterance, ingested the instant it's spoken.

recall

Auto-routed retrieval across the graph.

improve

Temporal cognify — the graph gets richer over time.

forget

Surgical, on command — “off the record.”

Cognee's API — add / search / cognify / prune — is already organized around exactly these four verbs. The hard product question arrives pre-answered; our job is wiring a live conversation into it fast enough to matter.

03The product

What Continuum does.

Listens to a live conversation, builds a knowledge graph of it as it's spoken, flags contradictions the instant they happen, and forgets on command.

06The four operations

Made visible — not decorative.

remember

cognee.add(dataset="continuum")

Every utterance, timestamped and speaker-tagged.

recall

cognee.search(GRAPH_COMPLETION | TEMPORAL)

Powers the Ask box and contradiction lookups.

improve

cognee.cognify(temporal_cognify=True)

Batched every 4 utterances + at end-of-call.

forget

cognee.prune.prune_data()

Wired to “off the record” — redacts both layers.

Every call into CogneeMemory flashes a status chip in the cockpit (_flash_op) — the four verbs Cognee exposes are a status light the user watches light up live, not an implementation detail buried in the backend.

Continuum07 / 12
07Groq: two models, two speeds

Fast enough to never fall behind the speaker.

llama-3.1-8b-instant

extract_facts(speaker, text)

Runs on every utterance. Pulls 0–3 structured facts — commitment, decision, number, risk — each with a canonical subject. Must return well under a second.

llama-3.3-70b-versatile

judge_contradiction() · answer_question()

Only runs when there's real reasoning to do — comparing a new fact against every prior fact on the same subject, or answering an open question.

Both calls use Groq's JSON mode, with a regex-rescue fallback if a response isn't clean JSON — the difference between a demo that silently recovers and one that crashes on stage.

Continuum08 / 12
08The knowledge graph

A graph you can walk into.

Every time the fact ledger changes, a small typed graph — speakers, subjects, facts, contradiction edges — is rebuilt. d3-force-3d runs a headless simulation (300 ticks); React Three Fiber renders the result as glowing, orbit-able spheres.

speakersubject commitmentdecision numbercontradiction

Color is functional, not decorative — it's the same taxonomy the fact extractor uses. When two facts contradict, both nodes get a coral halo and the edge renders dashed — the one visual cue in the whole cockpit that means stop and look here.

Continuum09 / 12
09Reliability

Never let the demo die.

A hackathon room has bad wifi, expiring keys, and installs that don't finish in time. Every layer of Continuum has a floor it can't fall through.

no key
regex + spoken-number extractor
no cognee
status → “degraded,” loop keeps running
bad json
regex-rescued from the raw response

Without GROQ_API_KEY, extraction falls back to hand-tuned regexes — including a spoken-number parser that turns “fifty five thousand” into 55000.0 with no LLM at all. The headline feature — live contradiction detection — never depends on either Groq or Cognee being present.

Continuum10 / 12
10What's next

The honest list, not the sales pitch.

  • Cross-session recall, surfaced. Every ended call is already archived with a summary — the UI to query a specific past session isn't built yet.
  • Temporal cognify at real scale. Proven on one scripted negotiation; the interesting failures only show up after a week of real calls.
  • Real speaker diarization. The mic path transcribes but doesn't yet distinguish speakers automatically.
  • Multi-party awareness. Track position and consensus across more than two speakers.

And what isn't built: auth, multi-instance WebSocket fan-out. None of it blocks the core loop — remember / recall / improve / forget and live contradiction detection are fully wired end to end, including with no external keys at all.
